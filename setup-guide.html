Portal Setup Guide

How to deploy your SOVATHA client portal online — step by step.

1 What you have

Three files in your portal folder:

  • client.html — what your clients see (upload, download, requests, messages)
  • admin.html — your private dashboard to manage all clients
  • setup-guide.html — this file

2 Host it free on GitHub Pages (recommended)

GitHub Pages hosts static HTML files for free with a real URL. No coding experience needed.

  1. Go to github.com and create a free account
  2. Click + New repository → name it sovatha-portal → set to Private
  3. Upload your three HTML files to the repo
  4. Go to Settings → Pages → Source → select main branch → Save
  5. Your portal is live at: https://yourusername.github.io/sovatha-portal/client.html
The repository is private so strangers can't find it, but the Pages URL works for anyone with the link — perfect for sharing with clients.

3 Generate client links

Each client gets a unique link based on a short code you assign them. Open your admin.html, click Add Client, enter their company name and a short code.

The link format is:

https://yourusername.github.io/sovatha-portal/client.html?c=CLIENTCODE

For example, for Khmer Angkor (kha):

https://yourusername.github.io/sovatha-portal/client.html?c=kha
Send this link to your client via Telegram, email, or WhatsApp. They bookmark it and use it every time.

4 Connect to Google Drive (for real file uploads)

Right now, uploads show in the portal UI but don't go anywhere. To connect to your real Google Drive, create a Google Apps Script Web App:

  1. Go to script.google.com → New project
  2. Paste the Apps Script code (see below)
  3. Deploy as Web App → Anyone can access → Copy the URL
  4. Paste the URL into client.html where it says YOUR_APPS_SCRIPT_URL
function doPost(e) {
  var data = JSON.parse(e.postData.contents);
  var clientCode = data.clientCode;
  var fileName = data.fileName;
  var fileData = data.fileData; // base64
  
  // Find the client's folder in Google Drive
  var parentId = "YOUR_MAIN_FOLDER_ID";
  var parent = DriveApp.getFolderById(parentId);
  var folders = parent.getFoldersByName(clientCode);
  
  var folder = folders.hasNext()
    ? folders.next()
    : parent.createFolder(clientCode);
  
  // Save the file
  var blob = Utilities.newBlob(
    Utilities.base64Decode(fileData), 
    data.mimeType, 
    fileName
  );
  folder.createFile(blob);
  
  // Email notification to you
  MailApp.sendEmail(
    "sovatha@consulting.com",
    "New file uploaded — " + clientCode,
    fileName + " was uploaded by " + clientCode
  );
  
  return ContentService.createTextOutput(
    JSON.stringify({success: true})
  ).setMimeType(ContentService.MimeType.JSON);
}

5 Keep your admin dashboard private

The admin.html file is for you only. Never share the admin link with clients. To add a password, you can use a free service like Netlify which supports password-protected pages, or simply keep the admin file locally on your computer and open it directly in your browser — it doesn't need to be hosted online to work.

6 Optional: custom domain

If you want a professional URL like portal.sovatha.com instead of a GitHub URL, you can:

  • Buy a domain from Namecheap (~$10/year)
  • Point it to your GitHub Pages site in the DNS settings
  • GitHub Pages supports custom domains for free
